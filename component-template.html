Article shell order (typical page): #article-fold → .article-content (body + ad/product slots) → #related-articles → #article-footer (reserved; often empty), all inside <section class="w-70 …"> so the related block stays the same width as the article column (do not close .w-70 before those two slots). Site header, nav, and site footer are injected by assets/js/main.js (defer). That script’s insertHeadElements injects a Google Fonts link for Playfair Display when no Playfair stylesheet is already in <head> (so the wordmark matches index.html / blog pages that declare fonts in markup). If the page has no link[rel="icon"] yet, it also inserts favicon tags and loads style.css + Bootstrap (see assets/js/main.js).

Injectors — createFold / createBreadcrumb (when breadcrumb: true); createProductShowcase, createProductCard (product card: green top bar + Playfair title in style.css); createComparisonTable + initComparisonTable; createAd; createTestimonialCarousel (also auto-inits [data-testimonial-carousel]). createArticleFooter() returns empty markup (slot reserved). Related articles: load assets/js/components/article/related-articles.js immediately after article-footer.js; it mounts on DOMContentLoaded into #related-articles using RELATED_CATALOG in that file—each row’s image should match the target article’s fold hero (imageSrc) or first in-content hero; resolveImageSrc() prefixes ../ for assets/… when the viewer is under /blog/. Optional on the slot: data-related-heading, data-related-count (1–6). Export: createRelatedArticlesHtml. Comparison table internal: createCell.
